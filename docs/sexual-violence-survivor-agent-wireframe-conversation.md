# 对话了解故事页 —— 交互设计草图

对应 [用户旅程图 · 阶段三](sexual-violence-survivor-agent-journey.md#阶段三对话了解故事)：用户选择嘉宾后进入对话页，通过问答了解该嘉宾的经历与观点；支持资源常驻、可随时返回列表或离开。

---

## 1. 设计目标与原则

| 目标 | 设计原则 |
|------|----------|
| 共情与理解 | 对话以嘉宾口吻回复，边界内呈现经历节点与态度；兜底话术温和，引导回已分享内容而非冷拒 |
| 支持资源常驻 | 「支持资源」入口常驻可见（如顶部或侧边），一键可达；用户感到不适时可快速找到出口 |
| 边界透明 | 问题超出档案时明确但温和地说明「不在分享范围内」，不展开、不批评提问者，保持信任 |
| 掌控感 | 用户可追问、换话题、随时返回嘉宾列表或离开；无自动跳转、无强制步骤 |

---

## 2. 信息架构（本页）

```
对话页（Conversation）
├── 顶部栏
│   ├── 返回列表 / 离开（明确出口）
│   ├── 当前嘉宾标识（化名 + 可选一句回顾，如「通过对话了解 ta 的故事」）
│   └── 支持资源入口（常驻）
├── 对话区（主内容）
│   ├── 消息列表（可滚动）
│   │   ├── Agent 回复气泡（嘉宾口吻）
│   │   ├── 用户问题气泡（可选展示）
│   │   └── 兜底/边界回复（如「这部分 ta 没有在分享范围内…」）
│   └── （可选）开场引导句（首条 Agent 消息，如「你可以问我你想了解的」）
├── 输入区（固定或吸底）
│   ├── 输入框（占位符提示可问方向）
│   └── 发送按钮（或 Enter 发送）
└── （可选）页脚：支持资源 · 返回列表 · 隐私说明
```

**优先级**：对话流可读、输入始终可用 > 支持资源与返回列表可见 > 嘉宾标识不抢戏。

## 4. 组件与交互说明

| 区域 | 组件类型 | 交互与状态 |
|------|----------|------------|
| 返回列表 | 链接/按钮 | 点击返回嘉宾列表页（阶段二），不弹确认；焦点顺序靠前，便于键盘/读屏 |
| 当前嘉宾标识 | 静态/轻量文案 | 化名 + 可选副标题「通过对话了解 ta 的故事」；不自动播放、不抢对话焦点 |
| 支持资源 | 入口链接/下拉/浮层 | 常驻于顶部栏；点击展开浮层或跳转支持资源页，内容与阶段一一致；可随时关闭 |
| 消息列表 | 滚动容器 | 新消息插入底部，自动滚动到底部（可选）；Agent 气泡与用户气泡视觉区分（左右或样式） |
| Agent 回复气泡 | 消息块 | 嘉宾口吻；若为兜底回复，可选用稍弱样式（如边框/背景区分）并配合温和引导文案 |
| 用户问题气泡 | 消息块（可选） | 展示用户刚发送的问题，便于上下文连贯；若产品选择「仅展示 Agent 回复」则可不展示 |
| 输入框 | 单行/多行输入 | 占位符提示可问方向（经历、态度、观点）；支持 Enter 发送；提交后清空，等待回复期间可禁用或显示「思考中」 |
| 发送按钮 | 主按钮 | 点击发送当前输入内容；无内容时禁用或隐藏 |
| 兜底/边界回复 | Agent 消息 | 当问题超出档案时触发；话术温和，说明「这部分 ta 没有在分享范围内」并引导回已分享主题，不批评用户 |

**焦点与顺序**：Tab 顺序建议「返回列表 → 支持资源 → 消息区（如有可聚焦元素）→ 输入框 → 发送」；读屏时按时间顺序朗读消息，再读输入区。

## 5. 文案要点（供内容与产品填写）

| 区块 | 要点 | 示例方向 |
|------|------|----------|
| 页面标题/嘉宾标识 | 明确当前对话对象，不暴露真实身份 | 「与『化名 A』对话」「通过问答了解 ta 的故事」 |
| Agent 首条/引导 | 邀请提问、说明边界在「愿意分享的范围内」 | 「你可以问我你想了解的，我会在愿意分享的范围内回答你。」 |
| 输入框占位符 | 提示可问方向，减少空白焦虑 | 「输入你想问的…（例如 ta 的经历、态度或观点）」 |
| 兜底话术 | 温和、不批评、引导回已分享内容 | 「这部分我暂时没有在分享里展开，你可以问问关于 [已分享主题示例]。」 |
| 支持资源浮层 | 与阶段一一致，强调可随时离开 | 「若您感到不适，可随时离开本页」+ 热线与机构链接 |

## 6. 与旅程的对应

| 旅程维度 | 本页设计响应 |
|----------|--------------|
| 用户目标「通过问答了解经历节点、态度与观点，形成共情与理解」 | 对话流为主内容；回复以嘉宾口吻、在边界内呈现；首条可引导提问方向 |
| 心声「我想问……」「这个回答是否来自 ta 真实分享？」 | 明确为「嘉宾口吻」的 Agent 回复；兜底时说明「未在分享范围内」，维持可信度 |
| 心声「有点难受时知道可以点支持资源」 | 支持资源常驻、一键可达；浮层内再次提示「可随时离开」 |
| 触点「常驻的支持资源入口；返回列表/离开」 | 顶部栏同时提供「返回列表」与「支持资源」，页脚可选重复 |
| 痛点「问题超出档案时只能得到不在分享范围内」 | 兜底话术温和、引导回已分享内容，避免冷拒感 |
| 痛点「用户自身有创伤时需快速看到支持资源」 | 支持资源入口不折叠、不依赖滚动，且浮层内提供热线等即时出口 |
| 机会点「回复始终在边界内，建立信任」 | 边界回复有统一、温和的文案与视觉处理，不突破档案 |
| 情绪「兜底打断时有轻微落差但理解是安全设计」 | 兜底样式与话术弱化「被拒」感，强调「引导」而非「禁止」 |

## 7. 可选设计变体

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| 不展示用户问题气泡 | 仅展示 Agent 回复，界面更简洁 | 若希望减少「对话感」或节省纵向空间 |
| 兜底回复单独样式 | 如左侧缩进、灰色边框、或「提示」图标 | 需要与正常回复区分、便于产品统计或迭代话术时 |
| 输入区多行可扩展 | 长问题可多行输入，发送前可换行 | 桌面端或希望减少误发时 |
| 「换一位嘉宾」入口 | 在顶部或输入区旁提供「换一位」直达列表 | 希望降低返回列表的认知成本时 |
