# 选择嘉宾页 —— 交互设计草图

对应 [用户旅程图 · 阶段二](sexual-violence-survivor-agent-journey.md#阶段二选择嘉宾)：用户同意进入后看到的嘉宾列表页，浏览 9 位嘉宾卡片、选择一位，可选再次触发警告后进入对话。

---

## 1. 设计目标与原则

| 目标 | 设计原则 |
|------|----------|
| 易选不累 | 9 位数量可控；一句话介绍突出「态度/主题」而非创伤细节；信息量适中，避免列表疲劳 |
| 掌控感 | 用户自主选择哪位嘉宾；再次触发警告可跳过但存在，不强制长读 |
| 安全延续 | 本页保留支持资源入口；进入对话前有一次简短提醒机会，维持创伤知情 |
| 认知清晰 | 卡片结构一致（化名 + 一句话 + 可选头像/剪影）；视觉层级明确，主操作「进入对话」可发现 |

---

## 2. 信息架构（本页）

```
选择嘉宾页（Guest List）
├── 页面标题（可选副标题）
├── 支持资源入口（常驻，与列表页同级）
├── 嘉宾列表（9 位）
│   └── 嘉宾卡片 × 9
│       ├── 化名/匿名
│       ├── 一句话介绍（态度/主题）
│       ├── 可选：头像/剪影
│       └── [进入对话] 按钮
└── （可选）再次触发警告
    └── 点击「进入对话」后：弹窗/浮层 简短提醒 + [仍要进入] / [再想想]
```

**优先级**：嘉宾卡片可扫视、可比较 > 支持资源可见 > 再次警告（轻量、可跳过）。

## 4. 组件与交互说明

| 区域 | 组件类型 | 交互与状态 |
|------|----------|------------|
| 页面标题 | 静态文案 | 可选副标题说明「通过对话了解 ta 的故事」；可提供「返回」回到阶段一（知情页），不记录离开原因 |
| 支持资源 | 入口链接/下拉 | 与列表页常驻；点击展开或跳转至支持资源页，与阶段一内容一致 |
| 嘉宾卡片 | 卡片容器 | 每卡：化名、一句话介绍、可选头像/剪影、主按钮；悬停/聚焦时可有轻量反馈（如边框或阴影），不自动展开更多内容 |
| 一句话介绍 | 文案（1 行或 2 行截断） | 突出态度/主题，避免创伤细节与刺激性措辞；字数建议约 15–30 字，超出可截断 + 省略号 |
| [进入对话] | 主按钮（每卡一个） | 点击 → 若启用「再次触发警告」则先弹出浮层；否则直接进入该嘉宾对话页。焦点顺序与读屏顺序按卡片自上而下、每卡内「进入对话」可聚焦 |
| 再次触发警告 | 弹窗/浮层 | 简短 2–3 句；「再想想」关闭弹窗回到列表；「进入对话」关闭弹窗并跳转该嘉宾对话页。可提供「下次不再提示」选项（仅当产品策略允许时） |
| 页脚 | 链接区 | 支持资源、返回首页、隐私说明等，按需放置 |

**焦点与顺序**：Tab 顺序建议为「支持资源 → 卡片 1 的进入对话 → 卡片 2 的进入对话 → …」；读屏时每张卡片作为一组（化名 + 介绍 + 按钮）朗读。

## 5. 文案要点（供内容填写）

| 区块 | 要点 | 示例方向 |
|------|------|----------|
| 页面标题/副标题 | 说明本页作用、降低选择压力 | 「选择一位嘉宾」「通过对话了解 ta 愿意分享的故事」 |
| 嘉宾一句话介绍 | 态度/主题，不写创伤细节 | 「更想谈谈如何与家人沟通」「关注司法程序中的感受」 |
| 再次触发警告 | 简短、提醒内容性质与出口 | 「对话内容可能涉及敏感话题。您可随时离开或使用支持资源。」 |
| 再想想 | 不施压 | 「再想想」或「返回列表」 |
| 进入对话（弹窗内） | 明确动作 | 「进入对话」与列表页按钮文案一致 |

## 6. 与旅程的对应

| 旅程维度 | 本页设计响应 |
|----------|--------------|
| 用户目标「找到想了解的那位或凭直觉选一位」 | 9 位卡片并列/网格，一句话介绍便于扫视与比较；主操作统一为「进入对话」 |
| 心声「哪位更想先了解？介绍是否足够让我做选择？」 | 一句话突出态度/主题；卡片结构一致，信息量可控 |
| 痛点「介绍过于模糊难以选择」 | 文案规范：态度/主题导向，避免空泛或过度隐晦 |
| 痛点「列表信息量过大易疲劳」 | 仅展示化名 + 一句话 + 可选头像；不在此页展开经历细节，保持 9 位 |
| 机会点「再次警告可跳过但存在」 | 弹窗/浮层简短提醒；提供「再想想」与「进入对话」，可选「下次不再提示」 |
| 情绪「选择时有掌控感」 | 无自动跳转、无默认选中；用户主动点击某卡的「进入对话」 |

---

## 7. 可选设计变体

| 变体 | 说明 | 适用场景 |
|------|------|----------|
| 无再次警告 | 点击「进入对话」直接进入对话页 | 希望流程更短、阶段一已充分告知时 |
| 再次警告仅首次 | 首次进入某嘉宾对话时弹窗，同一会话内不再弹 | 平衡提醒与流畅度 |
| 卡片可展开「更多介绍」 | 点击卡片或「更多」展开 2–3 句补充（仍不写创伤细节） | 若运营希望略多信息又避免列表过长 |
