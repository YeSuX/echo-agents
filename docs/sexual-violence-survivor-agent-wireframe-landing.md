# 接触与知情页 —— 交互设计草图

对应 [用户旅程图 · 阶段一](sexual-violence-survivor-agent-journey.md#阶段一接触与知情)：用户打开站点后看到的首屏，完成触发警告阅读、支持资源可见性、知情后选择进入或离开。

---

## 1. 设计目标与原则

| 目标 | 设计原则 |
|------|----------|
| 知情同意 | 触发警告与产品说明在一屏内可读完，不折叠核心信息；「同意进入」即表示已阅读并知情 |
| 支持资源可见 | 支持资源在首屏内可见且可点击，不依赖滚动或二次点击；视觉权重不低于主 CTA |
| 尊重选择 | 「进入」与「离开」并列，无默认焦点绑架；离开路径零成本（无挽留弹窗） |
| 控制认知负担 | 文案简短、分块清晰；避免长段落，用标题 + 短句列表 |

---

## 2. 信息架构（本页）

```
首屏（Landing）
├── 品牌/产品名（可选一句副标题）
├── 产品是什么（1–2 句）
├── 触发警告（Trigger Warning）
│   └── 简短说明内容可能涉及的主题与影响
├── 支持资源（常驻入口）
│   └── 热线 / 机构链接或展开列表
├── 知情与选择
│   ├── 说明：点击进入即表示已阅读上述内容
│   ├── [进入] 主按钮
│   └── [暂时离开] 次要按钮/链接
└── （可选）页脚：项目说明、隐私提示
```

**优先级**：支持资源 ≈ 触发警告 ≥ 产品说明 > 进入/离开按钮；按钮需在首屏内可见，无需滚动。

## 4. 组件与交互说明

| 区域 | 组件类型 | 交互与状态 |
|------|----------|------------|
| 产品名/副标题 | 静态文案 | 可选：副标题可链接到「关于我们」或项目说明页 |
| 产品说明 | 静态文案 | 纯阅读，不折叠 |
| 触发警告 | 区块（可带图标） | 不折叠；图标可用 ⚠ 或温和的警示图标，避免过于刺激 |
| 支持资源 | 标题 + 列表/手风琴 | 桌面：可与触发警告同一视觉层级，或置于其下独立区块；移动：可默认展开首条热线 + 「更多」展开；链接新开或站内页 |
| 「点击进入即表示…」 | 说明文案 | 紧贴主按钮上方，字体可略小、颜色次要 |
| [暂时离开] | 次要按钮或文字链接 | 点击：关闭页签 / 跳转至安全出口页（如搜索引擎或机构首页），不做挽留 |
| [进入] | 主按钮 | 点击：进入嘉宾列表页；无二次确认；可记录「已同意」用于合规（不涉及个人身份时仅记录为匿名同意） |

**焦点与顺序**：Tab 顺序建议为「支持资源链接 → 暂时离开 → 进入」，避免一上来就聚焦「进入」；读屏时先读触发警告与支持资源再读按钮。

## 5. 文案要点（供内容填写）

| 区块 | 要点 | 示例方向 |
|------|------|----------|
| 产品一句说明 | 做什么、谁的故事、如何呈现 | 「通过对话，了解九位嘉宾愿意分享的经历与观点」 |
| 触发警告 | 涉及什么主题、可能影响、可随时离开、支持资源在哪 | 「内容涉及性暴力与创伤相关叙述，可能引发强烈情绪。您可随时离开；如需支持请使用本页支持资源。」 |
| 支持资源 | 具体、可操作 | 热线号码、机构名+链接、可选一句「对话中也可随时查看」 |
| 知情说明 | 法律/伦理用途 | 「点击进入即表示您已阅读上述内容并知悉可能影响。」 |
| 暂时离开 | 不施压 | 「暂时离开」或「我稍后再看」，避免「取消」等负面措辞 |

---

## 6. 可访问性与安全

- **对比度**：触发警告与支持资源区块对比度满足 WCAG AA；支持资源链接使用焦点可见样式。
- **键盘与读屏**：支持资源、暂时离开、进入 均可键盘聚焦与激活；读屏时触发警告作为区域先被朗读。
- **安全出口**：「暂时离开」不记录任何分析；若使用「已同意」记录，仅用于合规且不关联可识别身份。
- **移动端**：支持资源在首屏内可见，避免需要大量滚动才看到热线。

## 7. 与旅程的对应

| 旅程维度 | 本页设计响应 |
|----------|--------------|
| 用户目标「知道是什么、有什么影响、哪里有求助」 | 产品说明 + 触发警告 + 支持资源同屏呈现 |
| 心声「感到不适怎么办？可随时退出吗？」 | 文案中明确「可随时离开」+ 支持资源醒目；离开按钮无挽留 |
| 痛点「警告过长易跳过」 | 触发警告控制在 2–4 句，不展开长文 |
| 痛点「资源不醒目」 | 支持资源独立区块、与触发警告同级或更显眼；移动端默认展开 |
| 机会点「同意即知情」 | 知情说明紧贴「进入」按钮，点击即视为知情 |
